# Getting Started

## What is OTI?

**OTI (OpenFlow Trust Infrastructure)** is a real-time, on-chain wallet trust scoring API. Send any wallet address and a blockchain — OTI returns a 0–100 trust score computed from five on-chain signals: wallet age, transaction count, token holding behavior, smart contract interactions, and transaction timing patterns. No ML model, no off-chain data — every number comes directly from verifiable on-chain activity.

OTI supports **15 blockchains** including Bitcoin, Ethereum, Solana, BNB Smart Chain, Tron, and TON.

## The OTI API

All requests go to a single base URL:

```
https://workspaceapi-server-production-5c0c.up.railway.app/api
```

:::info
This is the official, live OTI API — hosted on Railway (a cloud infrastructure provider). The URL is correct. A shorter branded domain is being set up and docs will be updated when it goes live.
:::

## Step 1 — Make Your First Request

No setup required. Paste this into your terminal and get a real score in seconds:

```bash
curl "https://workspaceapi-server-production-5c0c.up.railway.app/api/score/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045?chain=ethereum"
```

**Response:**

```json
{
  "address": "0xd8da6bf26964af9d7eed9e03e53415d37aa96045",
  "chain": "ethereum",
  "cached": false,
  "compromised": false,
  "score": 87,
  "signals": {
    "walletAge":                 { "score": 100, "weighted": 25, "maxWeight": 25 },
    "transactionCount":          { "score": 100, "weighted": 20, "maxWeight": 20 },
    "tokenHoldingBehavior":      { "score": 90,  "weighted": 18, "maxWeight": 20 },
    "smartContractInteractions": { "score": 85,  "weighted": 17, "maxWeight": 20 },
    "transactionTimingPatterns": { "score": 46,  "weighted": 7,  "maxWeight": 15 }
  }
}
```

`score` is the overall trust score (0–100). Each entry in `signals` shows the raw `score`, how many `weighted` points it contributed, and its `maxWeight` cap. See [Score Explanation](/score-explanation) for what each signal means.

:::tip Anonymous tier — free, no key needed
You get **100 requests per day** without any API key, per IP, resetting at midnight UTC. That is enough to build and fully test an integration.
:::

## Step 2 — Add Your API Key (for production)

When you're ready for production or need a higher daily limit, add your key to the `x-api-key` header:

```bash
curl "https://workspaceapi-server-production-5c0c.up.railway.app/api/score/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045?chain=ethereum" \
  -H "x-api-key: oti_your_key_here"
```

Keys follow the format `oti_<32 hex chars>` and are available on request at [openflowlabs.io](https://openflowlabs.io). Free-tier keys are available for low-volume and evaluation use.
